Hexo在GitHub建站

发表于 2018-05-22

Github + Hexo 并且通过Godaddy搭建静态个人网站详细教程（1）

首先上干货 王哲的Blog
开发环境：Windows 7
开发语言：Markdown
开发软件：Git，Node.js

作为一个程序员，有一个Blog来记录和分享平时所得是一个非常惬意的事情，但是由于种种原因，搭建一个私人的Blog是一件非常奢侈的事情，不仅成本高，而且很多时候繁琐的配置和环境已经吓退了一堆人了。
所以，就有了第二个选择，很多公司都提供了Blog业务，从远古时期的51Blog到如今的大名鼎鼎CSDN，博客园，都提供了非常成熟的Blog解决方案，但是用着现成的Blog，我还是不满足。虽然我技术不好，但是我的表现欲还是很足的，所以我还是希望能搭建一个自己的Blog。

直到我遇到Hexo。

Github，鼎鼎有名的程序员“同性”交友网站，应该是每个程序员心中的最神圣的地方。成千上万开源的精品项目在github这个温床里茁壮成长，同时也是程序员最”硬“实力的表现，别废话，你有多少颗Star。Github也提供了Pages功能，这就为了程序员打造一个免费的静态网页提供了最大的方便。Github还免费提供了二级域名，不过，这已经满足不了一个比较秀的程序员的欲望了，毕竟一个属于自己的域名才能在这芸芸众生中脱颖而出。

然后是Hexo，Hexo是一款基于Node.js的静态博客框架，可以生成静态页面而且可以托管于Github上。
下面开始从0开始搭建自己的Blog。

  • GitHub创建个人库
  • 安装Git
  • 安装Node.js
  • 安装Hexo
  • 推送个人网站
  • 绑定域名

Github创建个人库

首先，你可以注册一个Github账号，如果被墙了，请参阅各大VPN，翻墙网站，如果已经有了账号，请点击GitHub中的New repository创建新仓库，仓库名应该为：用户名.github.io, 这个用户名*使用你的GitHub帐号名称代替，这是固定写法，比如我的仓库名为：

joew4ng.github.io

安装Git

Git是开源的分布式版本控制系统，用于敏捷高效地处理项目。我们网站在本地搭建好了，需要使用Git同步到GitHub上。

Windows

Git是在Linux/Unix平台进行版本控制的利器，随着Windows用户的增多，越来越多的Windows生态圈的用户的需求增加，于是有大神就把Git打包移植到Windows平台。
Git)
然后傻瓜next下去就好了。

MacOS

在 Mac 上安装 Git 有多种方式。 最简单的方法是安装 Xcode Command Line Tools。 Mavericks （10.9） 或更高版本的系统中，在 Terminal 里尝试首次运行 git 命令即可。 如果没有安装过命令行开发者工具，将会提示你安装。

如果你想安装更新的版本，可以使用二进制安装程序。 官方维护的 OSX Git 安装程序可以在 Git 官方网站下载，网址为 http://git-scm.com/download/mac。

设置

安装完Git以后，然后就是设置Git了。
现在以在Windows下演示。

  1. 首先打开Git Bash

    git config —global user.name “你的GitHub用户名”
    git config —global user.email “你的GitHub注册邮箱”

  2. 生成ssh密钥文件

    ssh-keygen -t rsa -C “你的GitHub注册邮箱”
    后面出现的所有都是回车确定。
    然后找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容全部复制
    路径一般为 C:Users你电脑的名称/.ssh
    拷贝id_rsa.pub中的密钥

  3. 然后进入你的Github页面，进入SSH and GPG keys)页面，新建new SSH Key，标题任意填，个人建议填写Key连接的电脑，然后将刚才拷贝的id_rsa.pub信息粘贴进去，并点击确定。
  4. 回到 Git Bash并且输入

    ssh git@github.com
    成功即可，因为这样可以省下很多以后再次认证的时间，因为公钥在Github上，私钥在你的电脑中，因为Github每次推送都需要验证身份合法性，每次ssh的时候，Github会自动公钥和私钥这样以后你从你的开发电脑进行推送的时候会减去很多认证的时间。
    以上便是Git 的设置。

安装Node.js

Node.js 下载地址为Node.js Download), 请将添加环境变量勾选上。
安装完成以后输入：

node -v
如果出现安装的版本号，则安装成功。

安装Hexo

安装Hexo其实很简单，首先要在适当的位置新创建一个文件夹，以Blog Order 为例。
进入文件夹，按住shift并且点击鼠标右键，
点击 open command windows here
使用npm命令安装Hexo

npm install -g hero-cli
安心等待ing ………………………
安装完成后首先初始化Blog
hexo init blog
这样Blog就创建完毕了。
现在Blog还在本地，如果想先看看这个传说中的Hexo到底是什么样子的，可以现在本地进行测试，步骤如下。
hexo new hallo_welt #创建新的Markdown页面
hexo g #生成网页
hero s #启动服务预览

Hexo 命令小知识

如果没有什么错误的话，启动localhost:4000 就可以看到你刚才创建的Blog了。

推送个人网站

以上是Hexo创建并且初始化Blog的基本步骤，但是下面要做的就是将这个Blog推送到Github Pages中。
在Blog文件夹中有个_config.yml的文件，这个是站点配置文件。所有有关站点的配置都要在这个文件中进行。
首先需要将Hexo与Github关联起来，在_config.yml文件找到deploy并且配置如下。

deploy
type: git
repo: https://github.com/joew4ng/joew4ng.github.io.git
branch: master
但是经过我的测试，偶尔会发生识别不了repo，如果推送失败并且报错，请将repo改成
repo: git@github.com:joew4ng/joew4ng.github.io.git
亲测有效，修改完并且保存。
以上的步骤就是告诉Hexo需要把Blog部署到什么地方去。然后最后一步是需要安装Git部署的插件
npm install hexo-deployer-git —save
插件安装完成后，就需要将网站推送到你的Github中，其实很简单
hexo clean
hexo g
hero d
推送完成后，经过几分钟解析，输入你的Github地址比如 joew4ng.github.io便可以访问你的Blog了。

绑定域名

这个域名其实是我在和朋友聊天的时候无意脑子一抽就买了，运营商是Godaddy，以前没有用过这个运营商的域名，第一次尝尝鲜。
然后又没有钱去买独立的服务器，只能先绑定在Github上了。

  1. 首先你需要进入你的Blog setting页面，在这个页面中找到GitHub Pages，并且在Custom domain中填入你的私人域名。
  2. 进入你私人域名的DNS管理页面，增添三条记录即可。

    Type:A Name:@ Value:192.30.252.153
    Type:A Name:@ Value:192.30.252.154
    Type:CNAME Name:www Value:joew4ng.github.io
    经过一段时间的解析以后，通过你的私人域名就可以访问了。
    这就完成了Github和私人域名的绑定的功能了。
